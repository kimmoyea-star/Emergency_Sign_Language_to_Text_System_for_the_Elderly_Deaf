기존 응급 신고 체계의 한계 : 현재 112 및 119 신고 시스템은 음성 통화 위주이며, 문자/앱 신고 방식은 디지털 조작과 타자 입력에 익숙하지 않은 고령 농인에게 긴박한 상황에서 생명과 직결되는 높은 진입 장벽임. 또한 기존 영상 통화를 통한 수어 신고는 119 상황실 요원이 수어를 모를 경우 통역사를 거쳐야 하므로 골든타임이 지연되며, 통신 환경에 따라 영상이 끊기는 문제가 발생함. 고령 농인이 스마트폰 카메라에 핵심 응급 수어(가슴 통증, 화재 등)를 취하면, 기기 내부의 AI가 이를 즉시 텍스트로 번역하여 119 상황실 모니터에 전송하는 시스템을 제안함. 이를 통해 가벼운 텍스트 전송만으로 가장 빠르고 정확한 무장애(Barrier-free) 구조 요청 환경을 구축하고자 함.

### 개발 및 실행 방법
본 프로젝트는 VS Code와 Anaconda 환경에서 개발 및 실행함. 먼저 Anaconda를 이용하여 Python 3.10 환경을 구축한 후, 다음 라이브러리를 설치함.
* torch
* torchvision
* numpy
* pandas
* opencv-python
* mediapipe
* scikit-learn
* requests

데이터 수집 단계에서는 `data_collection.py`를 실행하여 웹캠을 통해 응급 수어 영상을 실시간으로 촬영하고, 이를 CSV 형식의 데이터로 저장함. 이후 `data_merge.py`를 사용하여 수집된 여러 개의 CSV 파일을 하나의 통합 데이터셋으로 병합함.

모델 학습 단계에서는 병합된 `data_all.csv` 파일을 로드하여 LSTM(Long Short-Term Memory) 기반 신경망을 학습시킴. 학습이 완료되면 최적의 가중치가 저장된 모델 파일인 `sign_model.pth`가 자동으로 생성 또는 업데이트됨.

실시간 인식 단계에서는 `telegram_119.py`를 사용함. 웹캠을 활성화하여 사용자의 수어 동작을 추적함. 인공지능 모델이 특정 수어를 80% 이상의 확신도로 인식하고, 해당 결과가1초 이상 유지될 경우 긴급 상황으로 판단함. 이후 외부 API를 통해 현재 위치 정보를 수집하고, 텔레그램 봇을 이용하여 상황실에게 긴급 알림 메시지를 자동 전송함.
정상적인 알림 수신을 위해서는 실행 전 `test_model.py` 파일 상단에 있는 `TELEGRAM_TOKEN`과 `TELEGRAM_CHAT_ID`에 사용자의 텔레그램 봇 정보를 입력해야 함.

프로그램 실행 중 웹캠 창을 선택한 상태에서 키보드의 ESC 키를 누르면 프로그램이 안전하게 종료됨.
